# 이커머스 도메인 요구사항 명세서

---

## 1. 개요

- **목적**: 좋아요·주문으로 이어지는 감성 이커머스의 핵심 거래 흐름(브랜드/상품/좋아요/주문)을 정의한다.
- **배경**: 여러 브랜드가 입점한 감성 이커머스 서비스를 가정한다. 방문자는 가입 없이 브랜드와 상품을 둘러볼 수 있고, 로그인한 사용자는 관심 상품에 좋아요를 눌러 두었다가 마음에 든 여러 상품을 한 번에 주문한다. 운영자(어드민)는 브랜드·상품을 등록·수정·삭제하고 전체 주문을 조회해 운영 현황을 파악한다. 좋아요와 주문이 맞물리는 이 거래 흐름이 서비스의 중심이다.

---

## 2. 이해관계자

| 구분 | 설명 | 식별 방식 | 권한 |
|-------|------|-----------|------|
| 비로그인 사용자 | 가입 없이 둘러보는 방문자 | 없음 | 브랜드·상품 **조회만** |
| 로그인 사용자 | 좋아요·주문을 하는 회원 | `X-Loopers-LoginId` / `X-Loopers-LoginPw` 헤더 | 조회 + 좋아요 + 주문, **본인 데이터만** |
| 어드민 | 브랜드·상품을 관리하는 운영자 | `X-Loopers-Ldap` 헤더 | 브랜드·상품 관리, **전체 주문 조회** |

---

## 3. 용어 정의

| 용어 | 정의 |
|------|------|
| 브랜드(Brand) | 상품을 묶는 판매 주체. 하나의 브랜드는 여러 상품을 가진다. |
| 상품(Product) | 판매 단위. 하나의 브랜드에 속한다. 가격과 재고를 가진다. |
| 재고(Stock) | 해당 상품을 지금 몇 개 더 팔 수 있는지를 나타내는 수량. |
| 좋아요(Like) | 사용자가 상품에 표시하는 관심. **한 사용자-상품 쌍당 최대 1개**. |
| 멱등(Idempotent) | 같은 요청을 여러 번 보내도 결과 상태가 동일한 성질. 좋아요 등록/취소에 적용. |
| 주문(Order) | 한 사용자가 한 번에 요청한 구매 단위. 한 개 이상의 주문 항목으로 이뤄진다. |
| 주문 항목(Order Item) | 주문에 담긴 상품 하나와 그 수량. |
| 주문 이력 | 주문 항목이 주문 시점의 상품명·단가를 복사해 보관한 값. 이후 상품 정보가 바뀌어도 이 값은 변하지 않는다. |

## 4. 기능 요구사항

> 각 기능은 "누가·무엇을·왜"를 한 문장으로 적고, 그 아래 **상세 동작 규칙**으로 구체 조건을 정의한다.

### 4.1 브랜드 · 상품 조회 (대고객)

#### US-01: 브랜드 정보 조회
방문자가 특정 브랜드의 정보를 조회해, 그 브랜드가 어떤 곳인지 확인하고 상품을 둘러볼 수 있게 한다.

**상세 동작 규칙**
- **AC-01-1**: 존재하는 브랜드를 조회하면, 브랜드의 고객 노출 정보(이름, 소개 등)를 보여준다.
- **AC-01-2**: 존재하지 않거나 삭제된 브랜드를 조회하면, "브랜드를 찾을 수 없음"이라고 안내한다.

#### US-02: 상품 목록 조회
방문자가 브랜드·정렬·페이지 조건으로 상품 목록을 조회해, 원하는 상품을 효율적으로 찾을 수 있게 한다.

**상세 동작 규칙**
- **AC-02-1**: 아무 조건 없이 목록을 요청하면, 기본 정렬(최신순)과 기본 페이지(첫 페이지, 20개)로 상품 목록을 보여준다.
- **AC-02-2**: 브랜드를 지정해 목록을 요청하면, 해당 브랜드의 상품만 보여준다.
- **AC-02-3**: 정렬 기준(최신순 / 가격 낮은순 / 좋아요 많은순)을 지정하면, 그 기준으로 정렬해 보여준다.
- **AC-02-4**: 조건에 맞는 상품이 하나도 없으면, 빈 목록을 정상적으로 보여준다.

#### US-03: 상품 상세 조회
방문자가 특정 상품의 상세 정보를 조회해, 구매 여부를 판단할 수 있게 한다.

**상세 동작 규칙**
- **AC-03-1**: 존재하는 상품을 조회하면, 상품명·가격·브랜드 정보·좋아요 수·판매 가능 여부를 보여준다.
- **AC-03-2**: 존재하지 않거나 삭제된 상품을 조회하면, "상품을 찾을 수 없음"이라고 안내한다.

---

### 4.2 좋아요 (대고객 · 멱등)

#### US-04: 상품 좋아요 등록
로그인 사용자가 마음에 드는 상품에 좋아요를 눌러, 나중에 다시 찾아보고 자신의 취향이 서비스에 반영되게 한다.

**상세 동작 규칙**
- **AC-04-1**: 아직 좋아요하지 않은 상품에 좋아요를 누르면, 좋아요가 등록되고 그 상품의 좋아요 수가 1 늘어난다.
- **AC-04-2**: **이미 좋아요한 상품**에 같은 요청을 다시 보내면, **오류 없이 성공으로 처리**하며 좋아요 수는 그대로 유지된다(멱등).
- **AC-04-3**: 존재하지 않는 상품에 좋아요를 누르면, "상품을 찾을 수 없음"이라고 안내한다.
- **AC-04-4**: 로그인하지 않은 상태에서 좋아요를 누르면, 요청을 거부한다.

#### US-05: 상품 좋아요 취소
로그인 사용자가 좋아요를 취소해, 더 이상 관심 없는 상품을 정리할 수 있게 한다.

**상세 동작 규칙**
- **AC-05-1**: 좋아요한 상품의 좋아요를 취소하면, 좋아요가 제거되고 그 상품의 좋아요 수가 1 줄어든다.
- **AC-05-2**: **좋아요하지 않은 상품**에 취소 요청을 보내면, **오류 없이 성공으로 처리**하며 좋아요 수는 그대로 유지된다(멱등).
- **AC-05-3**: 좋아요 수는 어떤 경우에도 음수가 되지 않는다.
- **AC-05-4**: **존재하지 않거나 삭제된 상품**에 취소 요청을 보내도, 그 상품에는 좋아요가 없으므로 **오류 없이 성공으로 처리**한다(AC-05-2와 동일하게 멱등). 좋아요 등록(AC-04-3)이 "상품을 찾을 수 없음"으로 거부하는 것과 의도적으로 다르다 — 취소는 "상품이 없으면 좋아요도 없다"는 관계라 좋아요 유무만으로 판정한다.

#### US-06: 내가 좋아요한 상품 목록 조회
로그인 사용자가 자신이 좋아요한 상품 목록을 조회해, 관심 상품을 한곳에서 확인할 수 있게 한다.

**상세 동작 규칙**
- **AC-06-1**: 로그인 사용자가 본인 좋아요 목록을 요청하면, 좋아요한 상품 목록을 페이징하여 보여준다.
- **AC-06-2**: 다른 사용자의 좋아요 목록을 요청하면, 요청을 거부한다(타 유저 정보 직접 접근 불가).

---

### 4.3 주문 (대고객)

#### US-07: 주문 생성
로그인 사용자가 여러 상품을 한 번에 주문해, 원하는 상품을 실제로 구매할 수 있게 한다.

**정상 흐름 요약**: 주문 요청 수신 → 상품·재고 확인 → 재고 차감 → 주문 생성(완료).

**상세 동작 규칙**
- **AC-07-1**: 주문 요청은 1개 이상의 주문 항목(상품·수량)으로 한다. 존재하지 않는 상품이 포함되거나 수량이 1 미만이면 주문을 거부한다.
- **AC-07-2**: 모든 상품의 재고가 충분하면, 각 상품 재고가 주문 수량만큼 차감되고 주문이 **완료** 상태로 생성된다.
- **AC-07-3**: 재고가 하나라도 부족하면, **주문은 생성되지 않으며**, 어떤 상품이 부족한지 안내한다.
- **AC-07-4**: 재고 차감과 주문 생성은 하나의 처리 단위로 묶인다 — 재고 부족으로 주문이 거부되면(AC-07-3) 재고는 변동되지 않는다.
- **AC-07-5**: 주문에는 주문 시점의 **상품명·단가가 이력으로 저장**되어, 이후 상품이 수정·삭제되어도 주문 상세 내역은 변하지 않는다.

> **[가정]** 한 주문에 동일 상품이 중복 포함되면 수량을 합산한다.

#### US-08: 내 주문 목록 조회
로그인 사용자가 기간을 지정해 자신의 주문 목록을 조회해, 과거 구매 내역을 확인할 수 있게 한다.

**상세 동작 규칙**
- **AC-08-1**: 조회 기간을 지정해 목록을 요청하면, 그 기간에 생성된 **본인 주문만** 최신순으로 보여준다.
- **AC-08-2**: 기간을 지정하지 않으면, 전체 기간의 주문을 보여준다.

#### US-09: 단일 주문 상세 조회
로그인 사용자가 특정 주문의 상세를 조회해, 무엇을 얼마에 샀는지 정확히 확인할 수 있게 한다.

**상세 동작 규칙**
- **AC-09-1**: 본인의 주문을 조회하면, 주문 항목별 이력(상품명·단가·수량), 총액, 주문 상태를 보여준다.
- **AC-09-2**: 다른 사용자의 주문을 조회하면, 요청을 거부한다.

---

### 4.4 브랜드 관리 (어드민)

#### US-10: 브랜드 등록
어드민이 새 브랜드를 등록해, 해당 브랜드의 상품을 판매할 수 있게 한다.

**상세 동작 규칙**
- **AC-10-1**: 필수 항목(브랜드명)을 갖춰 등록하면, 브랜드가 생성된다.
- **AC-10-2**: 필수값이 빠진 채 등록하면, 요청을 거부한다.

#### US-11: 브랜드 정보 수정
어드민이 기존 브랜드 정보를 수정해, 변경된 브랜드 정보를 반영할 수 있게 한다.

**상세 동작 규칙**
- **AC-11-1**: 존재하는 브랜드를 수정하면, 변경 내용이 반영된다.
- **AC-11-2**: 존재하지 않는 브랜드를 수정하면, 요청을 거부한다.

#### US-12: 브랜드 삭제
어드민이 브랜드를 삭제해, 더 이상 취급하지 않는 브랜드를 정리할 수 있게 한다.

**상세 동작 규칙**
- **AC-12-1**: 존재하는 브랜드를 삭제하면, **해당 브랜드의 모든 상품도 함께 삭제된다**.
- **AC-12-2**: 이미 생성된 주문은 이력으로 보존되므로, 브랜드·상품 삭제의 영향을 받지 않는다.
- **AC-12-3**: 브랜드·상품 삭제는 논리 삭제로 처리한다 — 데이터는 보존하되 조회·노출 대상에서 제외한다.

#### US-13: 브랜드 목록 / 상세 조회 (어드민)
어드민이 등록된 브랜드를 목록·상세로 조회해, 브랜드 현황을 관리할 수 있게 한다.

**상세 동작 규칙**
- **AC-13-1**: 어드민이 목록을 요청하면, 페이징된 브랜드 목록을 보여준다.

---

### 4.5 상품 관리 (어드민)

#### US-14: 상품 등록
어드민이 특정 브랜드에 상품을 등록해, 고객에게 판매할 수 있게 한다.

**상세 동작 규칙**
- **AC-14-1**: 필수 항목(상품명·가격·초기 재고)을 갖추고 **이미 등록된 브랜드**를 지정해 등록하면, 상품이 생성된다.
- **AC-14-2**: 존재하지 않는 브랜드에 상품을 등록하면, 요청을 거부한다.
- **AC-14-3**: 가격은 0 이상, 초기 재고는 0 이상의 정수여야 한다.

#### US-15: 상품 정보 수정
어드민이 상품 정보를 수정해, 가격·재고 등 변경 사항을 반영할 수 있게 한다.

**상세 동작 규칙**
- **AC-15-1**: 존재하는 상품을 수정하면, 변경 내용이 반영된다.
- **AC-15-2**: **상품의 브랜드는 수정할 수 없다.** 브랜드를 바꾸려는 요청은 거부한다.
- **AC-15-3**: 가격 수정은 **이후 주문부터** 적용되며, 기존 주문 이력에는 영향을 주지 않는다.
- **AC-15-4**: 재고 수정은 재고 수량을 **특정 값으로 설정**하는 것이다(증감이 아니라 절대 수량 반영).

#### US-16: 상품 삭제
어드민이 상품을 삭제해, 판매 종료된 상품을 정리할 수 있게 한다.

**상세 동작 규칙**
- **AC-16-1**: 존재하는 상품을 삭제하면, 고객 조회·주문 대상에서 제외된다.
- **AC-16-2**: 이미 주문에 포함된 상품은 이력 덕분에 주문 상세에 그대로 남는다.

#### US-17: 상품 목록 / 상세 조회 (어드민)
어드민이 등록된 상품을 브랜드별로 조회해, 재고를 관리할 수 있게 한다.

**상세 동작 규칙**
- **AC-17-1**: 어드민이 브랜드를 지정해 목록을 요청하면, 해당 브랜드 상품을 페이징하여 보여준다.

---

### 4.6 주문 조회 (어드민)

#### US-18: 주문 목록 / 상세 조회 (어드민)
어드민이 전체 주문을 목록·상세로 조회해, 운영 현황을 파악하고 문제를 추적할 수 있게 한다.

**상세 동작 규칙**
- **AC-18-1**: 어드민이 주문 목록을 요청하면, **모든 사용자의 주문**을 페이징하여 보여준다.
- **AC-18-2**: 어드민이 특정 주문을 조회하면, 주문자, 항목 이력, 총액, 주문 상태를 보여준다.

## 5. 비기능 요구사항

- **보안/권한**: **본인 데이터 접근 경계**(주문/좋아요 목록)는 명세상 강제한다.
